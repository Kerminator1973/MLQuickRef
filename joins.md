# Слияние, соединение и конкатенация

В концепции Pandas, DataFrame - это аналог таблиц в СУБД. Т.е. можно использовать колонки таблицы как внешнему ключу и с их помощью связывать DataFrame-ы между собой. Клю называется _внешним_, потому что егог значение хранится за пределами текущей таблицы.

При связывании таблиц могут встречатся расхождения по типам данных в колонках и часто следует явным образом указывать необходимый тип при загрузке CSV-файла:

```py
cities = pd.read_csv(
    "meetup/cities.csv", dtype = {"zip": "string"}
)
cities.head()
```

## Конкатенация

Конкатенация - это объединение двух таблиц с одинаковой структурой в одну таблицу. Например, есть один файл со списком жителей дома №14 и ещё один, с такой же структурой, но со списком жителей дома №16. Если мы хотим объединить всех жителей ээтих двух домов в одну таблицу, то мы выполняем конкатенацию.

Название функции конкатенации - `concat()`. Пример:

```py
pd.concat(objs = [groups1, groups2])
```

Pandas сохраняет индексные метки из оригинальных файлов, что может быть причиной коллиций id. Чтобы переделать индекс на новый, бесконфликтный, следует установить параметр ignore_index в значение True:

```py
pd.concat(objs = [groups1, groups2], ignore_index = True)
```

Если же мы хотим сохранить уникальные индексы таблиц, то необходимо добавить мультиндекс, в котором будет участвовать как оригинальный индекс, так и идентификатор таблицы при объединении. Второй идентификатор передаётся с помощью параметра `keys`:

```py
pd.concat(objs = [groups1, groups2], keys = ["G1", "G2"])
```

## Отсутствующие значения в объединённых DataFrame

При конкатенации DataFrame-ов в набор колонок может отличаться. Например, есть две похожие таблицы, в одной из которых есть колонка "Football", а в другой "Hockey". В общем случае, при конкатенации в результирующем DataFrame появятся обе колонки, но в для записей из разных таблиц часть ячеек будет содержать `NaN`.

Чтобы объединить наборы данных по оси столбцов, в функции concat() следует использовать дополнительный параметр `axis`:

```py
pd.concat(objs = [sports_champions_A, sports_champions_C], axis = 1)
```

либо:

```py
pd.concat(objs = [sports_champions_A, sports_champions_C], axis = "columns")
```

## Левые соединения (LEFT JOIN)

В отличие от конкатенации, операция соединения (join) связывает две таблицы по некоторому полю, которое есть в каждой из таблиц.

Левое соединение (LEFT JOIN) подходит для случаев, когда один набор находится в центре внимание, а второй используется для предоставляения дополнительной информации, связанной с первым набором.

Чтобы выполнить соединение следует использовать функцию merge():

```py
groups.merge(categories, how = "left", on = "category_id")
```
