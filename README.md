# Краткие заметки о машинном обучении

Задача данного репозитария - подготовить заметки, помогающие решать тестовые задачи на курсах обучения машинному обучению. В качестве языка программирования планируется использовать Python.

Основной инструмент работы - [Google Colab](https://colab.research.google.com/#), одно из ключевых преимуществ которого состоит в том, что он позволяет в удобном форме обмениваться "блокнотами" (.ipynb) - документами, которые совмещают в себе текстовые комментарии, формулы и код на Python, что делает его очень удобным при обмене данными между преподавателем и студентами. Также Colab позволяет избежать сложной настройки среды исполнения, автоматически доустанавливая необходимые для работы пакеты (packages).

У Google Colab отличная интеграция с Google Drive, которая позволяет сохранять .ipynb-файлы на Drive, и использовать Drive для запуска виртуальной машины Google Colab.

[В PyCharm появилась встроенная поддержка Google Colab](https://habr.com/ru/news/990942/) by daniilshat. Плагин Colab (от Google) для редактора кода VS Code публично доступен. С его помощью пользователи могут подключаться к удалённым ноутбукам и редактировать их локально.

## Основные задачи, решаемые с помощью ИИ

- Классификация - определить тип объекта (например, по фотографии, или видео - мужчина, или женщина)
- Регрессия - прогнозировать значения для объектов по некоторому набору свойств (например, доход человека по его возрасту, профессии, семейному положению, и т.д.). Линейная регрессия - степенная функция (одна из самых простых моделей)
- Кластеризация - группировка данных (например, распределить изображения разных букв по кластерам "вариант написания буквы")

Классификация и регрессия требуют обучающих данных, т.е. это обучение с "учителем". Для кластеризации обучающие данные не нужны, разделение на кластеры - алгоритмическое.

## Как решать любую задачу Data Science

Общая схема:

- Получить данные. Почистить (выбросы и шумы)
- Подготовить объекты и признаки
- Разделить данные на обучающую и тестовую выборки (при необходимости)
- Выбрать алгоритм машинного обучения
- Обучить модель на обучающей выборке
- Оценить качество обучения на тестовой выборке

## Эко-система ML на Python

**NumPy** (сокращение от Numerical Python) — это фундаментальная библиотека для языка программирования Python, предназначенная для эффективной работы с многомерными массивами и выполнения математических операций над ними. Она лежит в основе большинства инструментов для анализа данных, научных вычислений и машинного обучения в экосистеме Python.

Ключевые особенности:

- Объект ndarray (n‑dimensional array) — основной тип данных в NumPy:
    - многомерный массив фиксированного размера
    - все элементы имеют один и тот же тип данных (например, только целые числа или только числа с плавающей точкой)
    - данные хранятся в непрерывном блоке памяти — это ускоряет доступ и обработку
- Векторизация операций: позволяет применять математические операции ко всему массиву сразу, без использования циклов for
- Высокая производительность: библиотека написана на C и C++, поэтому операции выполняются в десятки и сотни раз быстрее, чем со стандартными списками Python
- Богатый набор математических функций:
    - арифметические операции
    - тригонометрические и логарифмические функции
    - статистические функции (среднее, дисперсия, медиана и т. д.)
    - операции линейной алгебры (умножение матриц, нахождение определителя, решение линейных уравнений и пр.)
    - быстрое преобразование Фурье (FFT)
- Бродкастинг (broadcasting): автоматическое расширение размерности массивов для выполнения операций с массивами разных форм без явного копирования данных.

**pandas** —  библиотека для языка программирования Python, созданная для обработки и анализа структурированных данных в табличном виде. Название происходит от термина Panel Data («панельные данные») — статистического понятия, обозначающего структурированные табличные данные.

Библиотека построена на базе NumPy и дополняет его функционалом для работы со сложными табличными структурами: разными типами данных, названиями столбцов и строк, пропущенными значениями и т. д.

Основные типы данных: Series и DataFrame. Основной источник данных - CSV-файлы.

>К недостаткам Pandas относят низкую стабильность при обработки огромных CSV-файлов. В качестве альтернативы рассматривается библиотека [Polars](https://pola.rs/). См. статью [Polars — "убийца Pandas" на максималках](https://habr.com/ru/articles/946788/) by alealandreev.

**scikit‑learn** (часто сокращают до sklearn) — это бесплатная библиотека с открытым исходным кодом для машинного обучения на языке Python. Она создана в 2007 году в рамках программы Google Summer of Code и с тех пор стала одним из ключевых инструментов в экосистеме Python для задач ML.

Библиотека написана на Python, C, C++ и Cython — это обеспечивает высокую производительность вычислений. Распространяется под лицензией BSD, что позволяет использовать её как в коммерческих, так и в открытых проектах.

scikit‑learn тесно интегрирована с другими библиотеками Python:

- NumPy — для работы с многомерными массивами
- SciPy — для научных вычислений
- pandas — для обработки и анализа данных
- matplotlib и seaborn — для визуализации результатов

Библиотека покрывает основные направления классического машинного обучения:

- Обучение с учителем (Supervised Learning):
    - классификация (определение категории объекта): логистическая регрессия, SVM, случайный лес, k‑ближайших соседей
    - регрессия (прогнозирование числового значения): линейная регрессия, деревья решений, градиентный бустинг
- Обучение без учителя (Unsupervised Learning):
    - кластеризация (группировка похожих объектов): k‑средних (k‑means), DBSCAN, иерархическая кластеризация
    - снижение размерности (уменьшение числа признаков): PCA (метод главных компонент), t‑SNE
- Предобработка данных:
    - масштабирование признаков (StandardScaler, MinMaxScaler)
    - кодирование категориальных переменных (OneHotEncoder, LabelEncoder)
    - работа с пропущенными значениями (SimpleImputer)
- Оценка и подбор моделей:
    - кросс‑валидация (cross_val_score)
    - подбор гиперпараметров (GridSearchCV, RandomizedSearchCV)

**Matplotlib** — фундаментальная библиотека для языка программирования Python, предназначенная для создания статических, анимированных и интерактивных визуализаций данных. Разработана Джоном Хантером в 2003 году как альтернатива графическим инструментам MATLAB.

Библиотека стала стандартом де‑факто для визуализации в экосистеме Python и лежит в основе многих других инструментов (например, seaborn, pandas).

**PyTorch** — открытый фреймворк для машинного и глубокого обучения на языке Python. Разработан командой искусственного интеллекта Meta на базе библиотеки Torch (для языка Lua). Первая версия выпущена в октябре 2016 года.

Ключевые возможности:

- Тензорные вычисления с поддержкой ускорения на GPU (по аналогии с NumPy, но с возможностью использования видеокарт через технологию CUDA)
- Динамические графы вычислений (Dynamic Computational Graph): структура нейросети определяется во время выполнения, а не заранее. Это упрощает отладку и эксперименты с архитектурами
- Автоматическое дифференцирование (модуль Autograd): автоматически вычисляет градиенты (производные) для обучения нейронных сетей — реализует алгоритм обратного распространения ошибки (backpropagation)
- Готовые модули для построения нейросетей:
    - nn — высокоуровневые абстракции для создания сложных архитектур
    -optim — алгоритмы оптимизации (SGD, Adam и др.)
    - инструменты для работы с данными, батчинг, аугментация
- Поддержка распределённого обучения на множестве GPU и серверов
    - DataParallel — простое распараллеливание
    - DistributedDataParallel — эффективное распределённое обучение
    - FSDP (Fully Sharded Data Parallel) — для очень больших моделей
- Экспорт моделей в формат TorchScript для развёртывания в production (например, через TorchServe)

Вокруг PyTorch сформировалось множество вспомогательных библиотек:

- PyTorch Lightning и Fast.ai — упрощают процесс обучения моделей
- Hugging Face Transformers — работа с предобученными моделями для NLP (BERT, GPT и др.)
- Pyro (от Uber) — вероятностное программирование
- Flair — обработка естественного языка
- Catalyst — обучение DL- и RL‑моделей

Однако следует отместить, что в репозитории PyTorch на GitHub можно увидеть [большое количество открытых issue](https://github.com/pytorch/pytorch/issues/). Это может свидетельствовать о том, что не все обращения пользователей обрабатываются своевременно. Среди открытых проблем встречаются как баги (ошибки в работе), так и запросы на новые функции, проблемы с установкой, ошибки в документации и другие.

Также отмечается нехватка контрибьюторов, что связано с:

- Высокая планка для новых функций (строгие требования maintainer-ов)
- Быстрый темп разработки
- Требования к навыкам контрибьюторов

Экосистема PyTorch является конкурентом TensorFlow (Google). PyTorch вырвался вперёд благодаря гибкости. Некоторые пользователи отмечают, что PyTorch менее эффективен для крупномасштабных моделей.

Экосистема Google TPU - закрытая. Считается, что она является т.н. _Vendor Lock_ - программные решения очень сильно сцеплены с аппаратными. Для программирования ML, у Google есть свой графовый компилятор XLA - Accelerated Linear Algebra.

>Проверил Google Colab: бесплатно можно подключаться к NVIDIA T4 (Turing) с 16 ГБ GDDR6 и Google TPU v5e-1 с 16 ГБ HBM2e. Платные тарифные планы предоставляют доступ к NVIDIA H100 c 80 ГБ HBM2e, NVIDIA G4 (Ampere?), NVIDIA A100 с 80 ГБ HBM2e и NVIDIA L4 (Ada Lovelace) c 24 ГБ GDDR6.

Графовые компиляторы обычно интегрированы в JAX, PyTorch и TensorFlow.

Рекомендуется почитать статью [Великий парадокс VRAM: почему мы платим миллионы за память, чтобы делать всё, лишь бы ей не пользоваться](https://habr.com/ru/articles/1036792/) by YH7H22.

**Keras (TensorFlow)** - открытый фреймворк для машинного и глубокого обучения от Google. Он позволяет строить и обучать модели, работать с тензорами (многомерными массивами), эффективно запускать вычисления как на CPU, так и на GPU/TPU. Это "движок" нижнего уровня: мощный, но довольно подробный в коде — приходится детально описывать граф вычислений.

Keras — это высокоуровневый API для построения нейронных сетей. Изначально был отдельной библиотекой, а сейчас официально встроен в TensorFlow как tf.keras.

Keras делает работу с нейросетями проще: вместо низкоуровневых операций модель описывается понятными блоками — слоями, функциями потерь, оптимизаторами. Например, создать простую сеть можно буквально в несколько строк кода.

[OpenCV](https://docs.opencv.org/4.x/d6/d00/tutorial_py_root.html) = Open Source Computer Vision Library — открытая библиотека с более чем 2 500 алгоритмами для решения задач компьютерного зрения и машинного обучения.

Основные возможности OpenCV:

- Обработка изображений:
    - изменение размера, обрезка, вращение;
    - фильтрация, шумоподавление;
    - преобразование цветового пространства (в оттенки серого, бинаризация и т. д.);
    - улучшение контраста, яркости.
- Распознавание объектов:
    - обнаружение лиц, глаз, дорожных знаков;
    - сегментация по цветам;
    - использование каскадов Хаара, HOG, SIFT, SURF и др.
- Анализ видео:
    - чтение и запись видеофайлов;
    - отслеживание движения;
    - анализ кадров в реальном времени.
- Машинное обучение:
    - встроенные модели для классификации;
    - интеграция с нейросетями (TensorFlow, PyTorch, ONNX) через модуль dnn.
- 3D‑реконструкция:
    - стереовидение;
    - оценка глубины;
    - построение 3D‑моделей.
- Работа с камерами:
    - захват видеопотока;
    - калибровка камер;
    - коррекция искажений.
- Рисование и визуализация:
    - добавление линий, фигур, текста на изображения;
    - подсветка обнаруженных объектов.

Для примера, для поиска окружностей на изображении можно использовать функцию `HoughCircles`(), а для поиска отдельных объектов сложной формы, может быть применена функция `connectedComponentsWithStats`().

**Tesseract OCR** (Optical Character Recognition) — открытая библиотека для распознавания текста на изображениях. Разработана компанией Hewlett‑Packard в 1980‑х, с 2006 года поддерживается Google и распространяется под лицензией Apache 2.0

Для работы с Tesseract в Python используется библиотека pytesseract — это обёртка над консольной утилитой Tesseract.

- Tesseract 4.x+: включает движок на основе рекуррентных нейронных сетей (LSTM), что значительно повысило точность распознавания.
- Tesseract 5.x: дальнейшие улучшения нейросетевого подхода, оптимизация производительности.

Чтобы улучшить результаты Tesseract, необходимо предварительно обработать изображение. Для этого используются следующие подходы:

- Увеличение разрешения: минимум 300 DPI
- Бинаризация: преобразование в чёрно‑белое изображение
- Удаление шума: фильтрация артефактов
- Выравнивание: коррекция перекоса текста
- Контрастность: улучшение различимости текста и фона

**YOLO** (_You Only Look Once_) — это алгоритм глубокого обучения, предназначенный для обнаружения объектов на изображениях и видео в режиме реального времени. Он был разработан в 2016 году Джозефом Редмоном и Али Фархади. Основная идея YOLO заключается в том, что изображение анализируется нейросетью за один проход, без разбиения процесса на отдельные этапы (например, поиск кандидатов и их последующую проверку). Это обеспечивает высокую скорость обработки данных.

- Основное назначение - обнаружение объектов: одновременное определение категорий и координат ограничивающих рамок нескольких объектов на изображении
- Используется глубокая нейронная сеть (свёрточная нейросеть, CNN), специализированная для задачи обнаружения объектов
- Может жертвовать частью точности ради скорости, особенно для маленьких объектов
- Рекомендуется использование GPU для достижения максимальной производительности; на CPU работает значительно медленнее

YOLO и OpenCV не являются прямыми конкурентами, так как решают разные задачи. YOLO специализируется на обнаружении объектов, а OpenCV — на общей обработке изображений и традиционных задачах компьютерного зрения. В многих проектах эти инструменты используются вместе: OpenCV применяется для предварительной обработки данных (например, шумоподавление, улучшение качества изображения), а YOLO — для обнаружения объектов.
