# scikit-learn

Официальный сайт [scikit-learn](https://scikit-learn.org/stable/)

Содержит большое количество разных моделей, включая линейную регрессию. Также содержит методы оценки качества, включая mse. Реализую методы разделения данных на выборки для обучения и выборку для валидации.

Типовой код:

```python
from sklearn.linear_model import LinearRegression
# Извлекаем какие-то данные из Dataset-а, который можно, например, скачать можно используя sklearn.datasets
# Присваиваем данные: X, y = ...
model = LinearRegression(fit_intercept=True)
model.fit(X, y)
prediction = model.predict(X)
```

В X находятся признаки, а в y - правильные ответы. Обучение модели на загруженных данных осуществляется с помощью вызова `fit()`. Выполнение предсказания осуществляется посредством вызова `predict()`.

Т.е. нам нужно:

- импортировать модель
- загрузить данные для обучения
- создать экземпляр модели
- провести обучение модели на данных
- используем обученную модель для получения результата

Признаки - это таблица (DataFrame), в которой каждая колонка - это некоторое свойство, а строки - конкретные объекты. Т.е. признаки - это таблица.

Правильные ответы - это результирующее значение, которое определяется по признакам. Когда мы говорим о y, мы имеем ввиду таблица из одной колонки, точнее даже не таблица, а серия (Series).

Разделение набора (split) на обучаемый и тестовый это просто разделение всех данных на две части:

```python
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
```

Довольно часто predict() используется для создания DataFrame-а, в котором первая колонка будет содержать реальное значение из оригинального dataset-а, в вторая колонка - рассчитанные, предсказанные данные. Такой Dataset нужен для того, чтобы к нему можно было применить алгоритм оценки точности модели. Например, может использоваться mean_squared_error (модуль metrics). Получая на входе Dataset с двумя колонками он вернёт квадрат средней ошибки (единственное значение).

Зачем нужны тестовые данные? Мы может применить predict() к тестовым данным и сравнить результаты с данными обучения. Если оба результата похожи, это означает, что модель не является переученной, т.е. она не "зазубрила" данные, а действительно научилась, т.е. нашла закономерности в предоставленных данных.
